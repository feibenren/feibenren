# 如何执行异步？
- 回调函数
- 事件监听
- 发布/订阅

# 回调函数
最常见的执行异步操作的方式
优点:最常见，最好理解
缺点:回调地狱,高度耦合,无法维护


# 事件监听
最常见的执行异步操作的方式
优点:好理解，去耦合
缺点:运行流程不清晰(比如，绑定了两个函数，哪个先执行都不知道，)


# 发布/订阅(观察者模式)
这个和事件监听很相似，但是比他高级
所有的程序运行权利都交由'信号中心'决定，可以从整体上把握整个程序的运行流程
优点:去耦合，运行流程清晰
缺点:不好理解
